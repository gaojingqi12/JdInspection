# 修复延期提测率

这份脚本基于桌面 `repair-delayed-launch` 的延期上线修复逻辑改造，用于处理 InsightEngine 中的延期提测明细。

## 处理逻辑

- 打开 `config.py` 中配置的延期提测率页面。
- 定位卡片 `需求明细-延期提测`。
- 筛选 `卡片进入测试阶段时间` 为上周五到今天。
- 筛选 `任务处理人部门C3` 为 `支付方案研发部`。
- 从表格中筛出：
  - `团队空间 = 支付生态研发部`
  - `是否延期提测 = 延期提测`
- 逐条点击需求名称跳转详情页。
- 在详情页把 `计划提测日期` 改成今天，并把处理结果写入 `history/YYYY-MM-DD.json`。

## 运行

```bash
python3 main.py
```

如需先只定位详情页的“周期与进度”卡片，不实际修改日期，可在 `config.py` 中设置：

```python
LOCATE_RELEASE_CARD_ONLY = True
```
