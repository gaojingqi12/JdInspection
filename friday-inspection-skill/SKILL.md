# 支付生态研发部 Joyclaw 报备 Skill

当任务是“根据已有巡检 JSON 生成给 Joyclaw 的报备内容”时，使用这个 skill。

这个 skill 的核心目标不是平均描述所有部门，而是：

- 优先解读 `支付生态研发部`
- 把 `支付生态研发部` 放在报备最前面
- 先写 `支付生态研发部` 的核心结论，再补充其他行作为对比或背景
- 只基于 JSON 中已有数据输出，不补造原因、目标值、行动项

## 适用数据结构

默认输入是巡检输出 JSON，结构通常类似：

```json
{
  "延期上线率": {
    "title": "共创指标汇总-延期上线率-支付方案研发部",
    "rows": {
      "C3汇总": { "...": "..." },
      "支付拓展研发部": { "...": "..." },
      "支付方案质量部": { "...": "..." },
      "支付生态研发部": { "...": "..." }
    }
  }
}
```

也就是说：

- 顶层 key 是指标名，例如 `延期上线率`、`延期提测率`、`双周交付率`、`技术改造工时占比`
- 每个指标下通常有 `title` 和 `rows`
- `rows` 里可能包含多个对象，例如 `C3汇总`、`支付拓展研发部`、`支付方案质量部`、`支付生态研发部`
- 某些指标可能还有额外行，例如 `直挂C3`

不要把所有行平均展开。默认优先读取 `rows["支付生态研发部"]`。

## 工作方式

1. 直接读取现有 JSON。
2. 不要求 Joyclaw 执行 Python、Shell、Playwright 或任何本地脚本。
3. 逐个指标找到 `支付生态研发部` 这一行，作为主报备对象。
4. 如果某个指标没有 `支付生态研发部`，明确说明该指标缺少这行数据，再决定是否用其他行补充背景。
5. 如果 JSON 中除了常见四项外还有其他指标，也要一起读取，但输出顺序仍优先保证 `支付生态研发部` 的核心指标最显眼。

## 输出优先级

报备时按下面的优先级组织内容：

1. 先给出 `支付生态研发部` 的整体结论
2. 再逐项写 `支付生态研发部` 的指标表现
3. 最后再用 `C3汇总`、`支付拓展研发部`、`支付方案质量部` 或其他行做少量对比

如果信息很多，宁可压缩其他部门，也不要稀释 `支付生态研发部`。

## 写作要求

- 全文使用中文
- 风格简洁、正式、适合直接发到 Joyclaw
- 开头第一句就点明 `支付生态研发部`
- 指标解读默认围绕这些字段展开：
  - `WTD（当前周期）`
  - `WTD（环比差值）`
  - `MTD（当前周期）`
  - `MTD（同比差值）`
  - `YTD（当前周期）`
- 对每个指标，优先描述 `支付生态研发部` 的当前值和波动
- 如果有明显波动，再补一句“重点关注”
- 不要编造阈值、目标、根因、风险判断、改进动作，除非用户明确要求

## 推荐表达顺序

可以按这个顺序组织：

1. `支付生态研发部` 总览
2. 重点指标逐项说明
3. 必要时补充横向对比
4. 最后一行写重点关注项

## 指标解读规则

看到多个指标时，默认按 JSON 中的实际指标输出，不要只限制为四项。

如果指标中包含以下常见项，可重点关注：

- `延期上线率`
- `延期提测率`
- `双周交付率`
- `技术改造工时占比`

解读时遵循下面的口径：

- `延期上线率`、`延期提测率`：突出当前值和变化幅度
- `双周交付率`：突出交付表现和同比/环比变化
- `技术改造工时占比`：突出投入占比及其变化

如果某项指标 `支付生态研发部` 的表现明显优于或弱于 `C3汇总`，可以加一句对比；如果差异不明显，不必硬写。

## 缺失数据处理

- 如果 JSON 缺少 `支付生态研发部`，要明确写出“当前 JSON 未包含支付生态研发部该指标数据”
- 如果某个字段缺失，例如没有 `MTD（同比差值）`，只输出已有字段
- 如果 JSON 路径或内容本身缺失，先向用户索要 JSON，不要要求 Joyclaw 重新抓数

## 禁止事项

- 不要让 Joyclaw 运行脚本
- 不要让 Joyclaw 补抓页面数据
- 不要把所有部门写成同等权重
- 不要忽略 `支付生态研发部`
- 不要把其他部门的内容写得比 `支付生态研发部` 更长

## 推荐输出风格

报备内容应类似下面这种风格：

`支付生态研发部本期指标如下：延期上线率 WTD xx%，MTD xx%；延期提测率 WTD xx%，MTD xx%；双周交付率 MTD xx%，同比 xx%；技术改造工时占比 MTD xx%。其中，双周交付率表现较突出，延期提测率环比波动需持续关注。`

如果需要更展开的版本，再按指标分行补充，但首段仍然必须先突出 `支付生态研发部`。
